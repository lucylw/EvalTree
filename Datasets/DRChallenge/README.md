# MyDataset (template / example)

Example dataset folder showing the exact files and formats EvalTree expects.
Copy this folder, rename it, and replace the contents with your real data.
See `docs/RUN_ON_NEW_DATASET.md` for the full walkthrough.

```
dataset.json                          # array of instances; here: {instruction, response}
splits/train-test.json                # OPTIONAL: 0-based indices into dataset.json (use --split train-test)
eval_results/real/my-model/results.json   # per-instance metric, SAME ORDER as dataset.json (accuracy: 0/1)
```

The `EvalTree/` subfolder (capabilities, embeddings, trees) is generated by the
pipeline stages — you do not create it by hand.
